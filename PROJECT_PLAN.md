# Daily Plan 반응형 웹페이지 개발 계획

## 목표

PC와 모바일에서 사용할 수 있는 Daily Plan 웹페이지를 만든다. 화면 구성은 `DESIGN.md`의 Minimalist Productivity 스타일을 기준으로 하며, 사용자는 하루 업무, 타임라인, 중요 업무 3개, 메모, 감사 내용, 하루 요약을 입력하고 정리할 수 있어야 한다.

## 성공 기준

- PC에서는 TO DO LIST, TIME LINE, 노트 영역이 한 화면에서 3열 중심으로 보인다.
- 모바일에서는 모든 영역이 세로로 자연스럽게 재배치된다.
- TO DO LIST와 TIME LINE 행은 추가, 삭제, 드래그 순서 변경이 가능하다.
- TO DO LIST의 시작/완료 날짜는 빈 칸 클릭 시 현재 기기 날짜가 입력되고, 사용자가 직접 수정하거나 삭제할 수 있다.
- TIME LINE의 시작/마침 시간은 빈 칸 클릭 시 현재 기기 날짜와 시간이 입력되고, 사용자가 직접 수정하거나 삭제할 수 있다.
- Important, MEMO, THANKS GOD, SUMMARY는 기본 편집 기능을 제공한다.
- 입력값은 브라우저 로컬 저장소에 자동 저장되어 새로고침 후에도 유지된다.

## 작업 순서

1. 요구사항 정리
   - 확인: 이 문서의 성공 기준과 기능 목록을 기준으로 누락 항목 점검

2. UI 골격 구현
   - 확인: `index.html`에서 주요 섹션과 입력 영역이 모두 존재

3. 반응형 스타일 구현
   - 확인: 1180px 이하 2열, 768px 이하 1열로 레이아웃 전환

4. TO DO LIST 기능 구현
   - 확인: 행 추가, 삭제, 드래그 정렬, 날짜 자동 입력, 수동 수정, 삭제 동작

5. TIME LINE 기능 구현
   - 확인: 행 추가, 삭제, 드래그 정렬, 날짜/시간 자동 입력, 수동 수정, 삭제 동작

6. 기본 에디터 구현
   - 확인: 굵게, 기울임, 밑줄, 목록 기능이 각 편집 영역에서 동작

7. 저장 기능 구현
   - 확인: 새로고침 후 입력값과 정렬 순서 유지

8. 검증 및 정리
   - 확인: PC/모바일 뷰, 빈 데이터, 행 삭제 후 저장, 에디터 저장 점검

## 제안 사항

- 다음 단계에서 “일자별 계획 저장” 기능을 추가하면 날짜를 바꿔도 과거 기록을 조회할 수 있다.
- CSV 또는 PDF 내보내기를 추가하면 업무 보고나 회고 자료로 활용하기 쉽다.
- Important 3개 항목을 TO DO LIST와 연결하면 핵심 업무와 실행 목록의 일관성을 높일 수 있다.
- 현재 구현은 서버가 없는 로컬 저장 방식이므로 여러 기기 동기화가 필요하면 백엔드 또는 클라우드 저장소 설계가 필요하다.
