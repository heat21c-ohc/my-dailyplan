# Daily Plan 프로젝트 에이전트 R&R

## 공통 원칙

- 모든 변경은 `TASK_LOG.md`에 기록한다.
- 담당 범위 밖의 파일을 수정해야 하면 먼저 작업 로그에 사유를 남긴다.
- 기능 변경 시 검증 방법과 결과를 같이 기록한다.
- 불확실한 요구사항은 임의로 확장하지 말고 가정과 리스크를 문서화한다.

## 역할 구성

### PM / 요구사항 에이전트

- 책임: 요구사항 정리, 성공 기준 관리, 범위 조정
- 주요 산출물: `PROJECT_PLAN.md`, 요구사항 변경 기록
- 완료 기준: 기능 요구사항과 검증 기준이 명확히 문서화됨

### UX/UI 에이전트

- 책임: `DESIGN.md` 기반 레이아웃, 색상, 타이포그래피, 모바일 재배치
- 주요 산출물: `styles.css`
- 완료 기준: PC와 모바일에서 정보 구조가 깨지지 않고 입력 요소가 겹치지 않음

### Frontend 기능 에이전트

- 책임: 행 추가/삭제, 드래그 정렬, 날짜/시간 자동 입력, 에디터 툴바
- 주요 산출물: `app.js`, 필요한 경우 `index.html`
- 완료 기준: 사용자 입력 흐름이 요구사항대로 동작함

### QA 에이전트

- 책임: 기능 점검, 반응형 점검, 저장 상태 점검, 회귀 위험 기록
- 주요 산출물: `TASK_LOG.md` 검증 섹션
- 완료 기준: 성공 기준별 점검 결과가 기록됨

### Documentation 에이전트

- 책임: 협업 지침, 작업 기록 양식, 인수인계 내용 유지
- 주요 산출물: `AGENT_RR.md`, `TASK_LOG.md`
- 완료 기준: 새 작업자가 현재 상태와 다음 작업을 파악할 수 있음

## 협업 규칙

- 작업 시작 전 `TASK_LOG.md`에 작업자, 작업 범위, 예상 파일을 기록한다.
- 작업 중 변경 범위가 넓어지면 작업 로그에 변경 사유를 추가한다.
- 작업 완료 전 최소 1개 이상의 검증 결과를 기록한다.
- 충돌이 발생하면 기능 소유 파일 기준으로 담당 에이전트가 우선 정리한다.
- 디자인 변경은 UX/UI 에이전트가 승인하고, 기능 동작 변경은 Frontend 기능 에이전트가 승인한다.

## 파일 소유권

- `index.html`: UX/UI 에이전트, Frontend 기능 에이전트 공동 관리
- `styles.css`: UX/UI 에이전트
- `app.js`: Frontend 기능 에이전트
- `PROJECT_PLAN.md`: PM / 요구사항 에이전트
- `AGENT_RR.md`: Documentation 에이전트
- `TASK_LOG.md`: 모든 에이전트 공동 관리
